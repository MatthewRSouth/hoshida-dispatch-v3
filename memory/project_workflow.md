User has a strict one-page-at-a-time workflow: build a page completely, then wait for review before starting the next page.

**Why:** User wants to review and approve each page before moving on.

**How to apply:** After finishing a page, stop and tell the user it's ready for review. Do not start the next page until explicitly asked.

Current build order:
1. Homepage (in progress as of 2026-03-25)
2. Teachers page
3. Schools page
4. Resources page (new)
5. Parent News page (new)
6. Parent Hub
